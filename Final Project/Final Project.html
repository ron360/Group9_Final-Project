Anomaly Detection

Identifying abnormal behaviors

Group9 許庭愷/李岳融/謝佳妤/陳厚宇/蔡清宇

Purpose:

  • Identify abnormal behavior or data
  • Recognize anomalies from normal behavior patterns
  • Enabling early issue detection and performance improvement

Origin and Evolution:

  • Early 20th century: quality control for manufacturing.
  • Early Methods: Relied on statistical and rule-based approaches.
  • Technological Advances: Principal Component Analysis (PCA) and Multivariate Control Charts (MCC).
  • Broadening application areas: finance, network security, healthcare, etc.

Statistics

  • Z score
  • Standard Deviation
    This method measures how far a data point is from the mean in terms of standard deviations.
  • InterQuartile Range
    Anomalies are detected by identifying data points outside the calculated IQR.

Machine Learning

  • Decision Tree: Random Forest
    Uses an ensemble of decision trees to identify anomalies in the data.
  • Supervised: Isolation Forest
    This algorithm builds an ensemble of isolation trees to isolate anomalies efficiently.
  • Deep Learning: Neural Network

  • Identify credit card fraud
    Abnormal detection can be applied to identify credit card fraud, helping financial institutions recognize irregular credit card transactions to safeguard customers' financial security. Supervised learning can be utilized to learn patterns of fraud and normal transactions from vast amounts of transaction data. Models can be trained based on known fraud cases.

    Alternatively, unsupervised learning methods can be employed to automatically discover abnormal patterns. Additionally, statistical methods such as mean, standard deviation, percentiles, ..., can be used to analyze users' historical transaction data. Transactions deviating significantly from the normal pattern can be flagged as abnormal. By leveraging these methods, financial institutions can build robust abnormal detection systems, enhancing the ability to detect credit card fraud while minimizing false positives and ensuring legitimate transactions are unaffected.

  • Industrial production
    Abnormal detection in the industrial production domain relies on advanced algorithms that leverage real-time data monitoring and analysis. These algorithms typically employ unsupervised learning, eliminating the need for specific defect data, and instead undergo pre-training using normal operational data.

    By comparing real-time production data, the algorithms can identify abnormal patterns or values, indicating potential issues. This technology contributes to improving production line efficiency, optimizing quality, and simultaneously reducing failure rates.

    Furthermore, it facilitates the implementation of predictive maintenance, allowing businesses to proactively prevent machine failures and minimize downtime. In summary, abnormal detection provides powerful tools for industrial production, enabling the realization of a more reliable, efficient, and sustainable manufacturing environment.

  • Assistance in account security
    The application of anomaly detection in account security can provide significant assistance in reducing some cybersecurity risks. By collecting user behaviors and authentication activities, such as login times, IP addresses, used devices, and performed operations, it is possible to establish a model of normal user behavior.

    Subsequently, this model can be applied to real-time data to detect abnormal behaviors. The use of anomaly detection can help prevent abnormal activities, such as hacking attempts, by detecting issues before they cause actual harm, thereby proactively identifying potential security risks. The model can also be updated based on new data and behavior patterns to adapt to changes in system and user behavior.

Pros:

  • Early detection
    Anomaly detection is effective in identifying unusual patterns or behaviors in data, allowing for early detection of potential issues or threats.
  • Automation
    Many anomaly detection algorithms can operate in real-time or near real-time, allowing for automated and timely responses to anomalies.
  • Adaptability
    Anomaly detection methods can adapt to changes in the data distribution over time, making them suitable for dynamic environments.


Future:


With the continuous advancement of artificial intelligence deep learning technology, there may be more complex and refined deep learning models in the future, which can enhance the accuracy and efficiency of anomaly detection.

Anomaly detection systems may also become more autonomous, capable of automatically learning and adapting to new anomaly patterns, reducing the need for human intervention. As contemporary society demands increasing real-time capabilities, future anomaly detection systems may place greater emphasis on real-time performance to promptly respond and detect anomalies.

Cons:

  • Difficulty in Defining Normal Behavior
    Determining what constitutes normal behavior can be subjective and challenging, especially in complex systems where normal patterns may evolve over time.
  • Dependency on Data Quality
    Anomaly detection relies heavily on the quality of input data. Noisy or incomplete data can lead to inaccurate results.
